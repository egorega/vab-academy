# Инструкция: Как закинуть этот проект на GitHub (Первый запуск)

Этот файл читает тот человек, который сейчас держит всю папку `vab-academy` у себя на компьютере (создатель проекта).
Прямо сейчас все эти уроки лежат только у вас на жестком диске. Чтобы команда увидела их, вам нужно отправить их в облако.

**Сделайте это прямо сейчас (до старта Дня 1):**

### Шаг 1: Создайте пустой сейф в облаке
1. Зайдите на сайт [github.com](https://github.com) и войдите в свой аккаунт.
2. В правом верхнем углу нажмите плюсик `+` -> **New repository**.
3. В поле *Repository name* напишите: `vab-academy`
4. Выберите **Public** (чтобы команда могла читать без проблем) или **Private** (если хотите скрыть код от всего интернета, но тогда вам придется вручную добавить email-адреса всей команды в настройках репозитория).
5. **ВНИМАНИЕ:** Не ставьте галочку "Add a README file"! Репозиторий должен быть абсолютно пустым.
6. Нажмите зеленую кнопку **Create repository**.

### Шаг 2: Отправка файлов с вашего компьютера
Откройте Терминал (PowerShell или обычную командную строку) и перейдите в папку, где лежат эти файлы (папка `vab-academy`).
Затем скопируйте и вставляйте эти команды по одной:

1. Создаем локальный Git-репозиторий:
```bash
git init
```

2. Добавляем все папки (Curriculum, README) в корзину для отправки:
```bash
git add .
```

3. "Фотографируем" эти файлы:
```bash
git commit -m "Initial commit: VAB Academy Curriculum Day 1-7"
```

4. Переименовываем главную ветку в main (стандарт индустрии):
```bash
git branch -M main
```

5. Привязываем вашу локальную папку к облаку (Замените `ВАШ_НИКНЕЙМ` на ваш логин в GitHub):
```bash
git remote add origin https://github.com/ВАШ_НИКНЕЙМ/vab-academy.git
```

6. Отправляем файлы в облако:
```bash
git push -u origin main
```

### Шаг 3: Раздача доступов
Всё! Теперь скиньте ссылку на ваш GitHub репозиторий в чат команды (например, `https://github.com/Ega/vab-academy`). 

С этого момента команда может заходить туда, читать `README.md`, а на 3-й день — склонировать его себе на компьютеры с помощью команды `git pull`.
