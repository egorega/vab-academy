## 🧠 Теория (Глубокое погружение)

### 1. Терминал (Командная строка)
Обычные люди общаются с компьютером через GUI (Графический интерфейс — папки, иконки, мышка). Это медленно и ограничено. Инженеры используют **Терминал (CLI)**. 
Вы пишите текстовую команду, и компьютер мгновенно её выполняет. 

Самые важные команды (одинаковы почти везде):
- `pwd` (Print Working Directory) — "Где я сейчас нахожусь?"
- `ls` (List) — "Что лежит в этой папке?"
- `cd папка` (Change Directory) — "Зайти в папку".
- `cd ..` — "Выйти на уровень назад".
- `mkdir имя` (Make Directory) — "Создать папку".

### 2. Git: Машина времени для кода
Представьте, что вы пишете курсовую. У вас появляются файлы: `Курсач.docx`, `Курсач_финал.docx`, `Курсач_Точно_Финал_2.docx`. Это хаос.
Для кода придумали **Git**. Это программа, которая следит за изменениями в ваших файлах.

- **Репозиторий (Repo):** Это просто папка на вашем компьютере, за которой следит Git.
- **Коммит (Commit):** Это "фотография" всей вашей папки в определенный момент времени. Вы написали код -> сделали коммит. Если завтра всё сломается, вы одной командой откатитесь к вчерашней "фотографии".

### 3. Три состояния файла в Git
Git не сохраняет всё автоматически. Вы должны сказать ему, что сохранять:
1. **Modified (Изменен):** Вы поменяли файл, но Git просто заметил это.
2. **Staged (Подготовлен):** Вы сказали Git: "Обрати внимание на этот файл, я скоро сделаю его фотографию" (команда `git add`).
3. **Committed (Сохранен):** Вы сделали фотографию (команда `git commit -m "Сообщение"`).

## 🛠️ Практика (Задание на сегодня)

**Par (Телефонный режим):** Просто прочитайте теорию выше и посмотрите видео "Git commit explained". **НО 25 июля**, как только вы сядете за ПК, вы обязаны вернуться в этот файл и выполнить практические шаги ниже, иначе вы не сможете пройти День 3!
**Остальные (Практика за ПК):**

1. Откройте Терминал (PowerShell в Windows или Terminal в macOS).
2. Напишите `pwd`, чтобы понять, где вы.
3. Напишите `mkdir VAB_Test` и затем зайдите туда `cd VAB_Test`.
4. Напишите `git init` (Это скажет Git: "Начни следить за этой папкой").
5. Создайте пустой текстовый файл. (В терминале: `echo "Hello" > test.txt`).
6. Напишите `git status` (Вы увидите красную надпись — файл не отслеживается).
7. Подготовьте файл к коммиту: `git add test.txt`.
8. Напишите `git status` (Текст стал зеленым — файл в стадии Staged).
9. Сделайте "фотографию": `git commit -m "Мой первый коммит"`.
