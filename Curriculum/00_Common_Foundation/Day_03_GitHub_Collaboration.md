## 🧠 Теория (Глубокое погружение)

### 1. Git vs GitHub
Частая ошибка новичков — путать эти два понятия.
- **Git:** Это бесплатная программа *на вашем компьютере*. Она делает коммиты.
- **GitHub:** Это коммерческий *сайт в интернете* (принадлежит Microsoft). Это облачный диск для ваших репозиториев.

### 2. Ветки (Branches)
Представьте, что мы пишем книгу. Ega пишет главу 1, а Par главу 2. Если мы будем редактировать один и тот же файл одновременно, мы всё сломаем.
В Git есть **Ветки**. 
Главный ствол называется `main` (или `master`). Это рабочий продукт.
Когда Ega хочет добавить фичу, он создает свою "копию" вселенной: ветку `feature/login`. Он работает там. Когда всё готово, он сливает свою ветку обратно в `main`.

### 3. Pull Request (PR) и Кросс-ревью
В профессиональных командах вы не можете просто так запихнуть свой код в `main`. 
Вы отправляете ветку на GitHub и нажимаете кнопку **Open Pull Request**. 
Это означает: *"Команда, я написал код. Пожалуйста, посмотрите его, и если всё ок, разрешите мне слить его с главной версией"*.

**Важнейшие правила из нашего README:**
1. **Запрет на само-аппрув:** Вы КАТЕГОРИЧЕСКИ не можете одобрять (Approve) свой собственный PR. Код без проверки — это мертвый код.
2. **Кросс-ревью:** Если ваш напарник оффлайн (например, Ega уехал), ваш код проверяет инженер другой специализации (например, Фронтендер Par). Вы обязаны так описать свой PR, чтобы даже человек из другой сферы понял логику.
3. **Щит QA:** Ваш код не попадет в `main`, пока тестировщик (Joe) не даст свой Аппрув, проверив функционал.

### 4. Как живет наш План обучения? (ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ)
Тот самый текст, который вы сейчас читаете — это не файл в Telegram. Это часть кода нашего репозитория. 
Если завтра мы решим добавить "День 25" или изменить правила в `README.md`, мы не будем пересылать вам новый файл.
Кто-то внесет изменения и сделает `git push` в ветку `main`.
Чтобы увидеть эти новые правила или уроки на своем компьютере, **вы обязаны каждое утро перед началом работы прописывать команду `git pull`**. Это синхронизирует ваш локальный компьютер с последней версией "Облака".

## 🛠️ Практика (Задание на сегодня)

**Raf (Менеджер с телефона):** Изучите интерфейс GitHub Mobile. Найдите раздел Pull Requests в любом открытом репозитории.
**Остальные (Практика за ПК):**

Мы будем использовать репозиторий `vab-academy`, в котором вы сейчас находитесь.
1. Откройте Терминал в папке `vab-academy`.
2. Скачайте последние изменения с сервера: `git pull origin main`.
3. Создайте свою личную ветку: `git checkout -b student/ваше_имя` (Например: `git checkout -b student/ega`).
4. Зайдите в папку `00_Common_Foundation/`.
5. Создайте текстовый файл `ваше_имя.txt` (Например `ega.txt`) и напишите внутри: "Я прошел общую базу".
6. Закоммитьте изменения: 
   - `git add .`
   - `git commit -m "Add my intro file"`
7. Отправьте ветку на GitHub: `git push origin student/ваше_имя`
8. Зайдите на сайт GitHub в репозиторий `vab-academy`. Вы увидите зеленую кнопку "Compare & pull request". Нажмите её!
