## 🧠 Теория (Глубокое погружение)

### 1. Что такое Класс?
Класс — это чертеж. Например, чертеж автомобиля. По чертежу нельзя ездить, но в нем описано, что у машины есть 4 колеса и цвет.
Объект (Экземпляр) — это конкретная машина, собранная на заводе по этому чертежу. Красная Toyota и Синий Ford — это два разных **объекта** одного **класса** Автомобиль.

```python
class User:
    # __init__ — это конструктор. Он вызывается в момент создания объекта.
    def __init__(self, username, role):
        self.username = username # self — это указание на КОНКРЕТНЫЙ объект
        self.role = role

    # Это метод (функция внутри класса)
    def say_hello(self):
        print(f"Привет, я {self.username}, моя роль: {self.role}")

# Создаем конкретных людей (объекты)
user1 = User("Ega", "Backend")
user2 = User("Joe", "QA")

user1.say_hello()
user2.say_hello()
```

### 2. Зачем нужен `self`?
Если вы пишете метод внутри класса, первым аргументом всегда должен быть `self`. 
Когда вы вызываете `user1.say_hello()`, Python невидимо передает `user1` внутрь метода в качестве `self`. Так программа понимает, ЧЬЁ именно имя нужно напечатать.

### 3. Инкапсуляция (Скрытие данных)
Хороший тон в ООП — не давать никому менять данные объекта напрямую.
Вместо того чтобы делать `user1.balance = 1000000` (так любой хакер сломает систему), мы пишем методы: `user1.deposit(500)`.
