## 🧠 Теория (Глубокое погружение)

### 1. Функции (DRY - Don't Repeat Yourself)
Главное правило программиста — "Не повторяйся". Если вы копируете один и тот же код дважды, вы делаете что-то не так. Оберните его в функцию.
```python
def greet_user(username, age):
    print(f"Привет, {username}, тебе {age} лет.")

greet_user("Hip", 16) # Вызов функции
```

### 2. Scope (Область видимости переменных)
Это самая частая причина странных багов.
- **Глобальная переменная:** Создана вне всех функций. Её видят все, но изменять её внутри функции — **очень плохая идея** (нужно ключевое слово `global`).
- **Локальная переменная:** Создана внутри функции. Как только функция заканчивает работу, переменная **уничтожается**.

```python
x = 10 # Глобальная

def my_func():
    y = 5 # Локальная
    print(x) # Сработает (10)

my_func()
print(y) # ОШИБКА! Переменная y мертва.
```

### 3. Чистые функции (Pure Functions)
Чистая функция — это святой грааль инженерии. Это функция, которая:
1. Зависит только от своих аргументов (не читает глобальные переменные).
2. Ничего не ломает вокруг (не меняет глобальные переменные, не пишет в файлы).
3. Просто берет данные на вход `->` возвращает результат (через `return`).
Такие функции невероятно легко тестировать (привет, Joe!).

*Объединяем функции и циклы! Имитируем реальную работу с данными.*
Напиши функцию `analyze_prices(price_list)`, которая принимает список чисел (цен). 
Внутри функции используй цикл `for`, чтобы найти: самую высокую цену, самую низкую цену и среднее арифметическое.
