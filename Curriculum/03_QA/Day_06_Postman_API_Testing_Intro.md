## 🧠 Теория (Глубокое погружение)

### 1. Что такое API? (Application Programming Interface)
Представь, что ты пришел в ресторан. 
- Ты (Клиент / Frontend Par). 
- Повар на кухне (Сервер / Backend Hip, Ega).
- Меню и Официант — это **API**. 

Ты не можешь сам зайти на кухню и приготовить бургер (Frontend не имеет прямого доступа к базе данных). Ты даешь официанту (API) точный запрос (Request): "Хочу бургер, без лука". Официант уносит заказ повару (Backend) и приносит тебе ответ (Response).

### 2. Запросы (Requests) и Ответы (Responses)
В вебе официанты общаются по протоколу HTTP. 
У каждого запроса есть **Метод (Действие)**:
- `GET` — дай мне данные (Принеси бургер).
- `POST` — создай новые данные (Регистрация нового пользователя).
- `PUT / PATCH` — обнови данные (Поменять пароль).
- `DELETE` — удали данные.

У ответа сервера всегда есть **Статус Код**:
- `200 OK` — всё прошло успешно.
- `400 Bad Request` — ты (клиент) прислал кривой запрос.
- `404 Not Found` — такого адреса нет.
- `500 Internal Server Error` — сервер (код Бэкенда) сломался.

### 3. Формат JSON
Люди общаются на русском или английском. Программы общаются на **JSON** (JavaScript Object Notation). Это просто текстовый формат передачи данных.
```json
{
  "user_id": 105,
  "name": "Joe",
  "role": "QA",
  "is_active": true
}
```
