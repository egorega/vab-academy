## 🧠 Теория (Глубокое погружение)

### 1. Media Queries (Медиазапросы)
CSS позволяет применять стили только при определенных условиях (например, если экран меньше 768 пикселей шириной).
```css
/* Базовые стили для десктопа */
.container {
    display: flex;
    flex-direction: row; /* Элементы в ряд */
}

/* Стили только для телефонов */
@media (max-width: 768px) {
    .container {
        flex-direction: column; /* Элементы друг под другом */
    }
}
```

### 2. Подход Mobile First (Сначала мобильные)
Раньше сайты верстали для больших экранов, а потом "сжимали" под телефоны. Это плохая практика, потому что телефон качает тяжелые стили десктопа.
Современный подход — **Mobile First**. Сначала мы пишем базовые стили так, будто сайт откроют на телефоне (в колонку). А потом через `@media (min-width: 1024px)` добавляем стили для десктопов (раскидываем по сетке Grid).

### 3. Transitions и Animations (Микроинтеракции)
Сайт без анимаций кажется мертвым. 
- **Transition:** Плавное изменение свойства. Например, кнопка меняет цвет при наведении (hover) не резко, а за 0.3 секунды.
- **Animation (Keyframes):** Сложные сценарии анимации. Например, элемент плавно выезжает слева при загрузке страницы.

```css
button {
    background-color: blue;
    transition: background-color 0.3s ease-in-out; /* Плавность */
}
button:hover {
    background-color: red; /* При наведении станет красной за 0.3 сек */
}
```
